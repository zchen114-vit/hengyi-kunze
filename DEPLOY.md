# 恆易 · 坤澤 - 部署與發佈指南

本文說明如何讓別人使用這個 App，從本地執行到真正上架。

## 1. 本地執行（自己用）

```bash
cd D:\Grok\hengyi-kunze
pip install -r requirements.txt
streamlit run app.py
```

- 會自動開啟瀏覽器。
- 手機也可以連同一個 Wi-Fi 下的網址使用（看 terminal 顯示的網址）。

---

## 2. 最簡單讓大家使用：部署到網路上（強烈推薦先做這步）

### 方法 A: Streamlit Community Cloud（免費、最快，建議先做這個）

這是目前把 Streamlit App 給別人用的**最簡單**方式。

#### 詳細步驟：

1. **先把程式上傳到 GitHub**
   - 去 [github.com](https://github.com) 建立新 Repository（建議取名 `hengyi-kunze`）
   - 把這個資料夾用 Git 上傳（或用 GitHub Desktop）

2. **部署到 Streamlit Cloud**
   - 打開 https://share.streamlit.io
   - 用 GitHub 帳號登入
   - 點右上角 **New app**
   - 選擇你剛剛的 GitHub repo
   - 主檔案保持 `app.py`
   - 點 **Deploy!**

3. 部署成功後，會得到一個類似 `https://yourname-hengyi-kunze.streamlit.app` 的網址。

4. 把這個網址分享給別人即可。

#### 手機使用方式（加到桌面）：
- iPhone：用 Safari 開網址 → 分享按鈕 → 「加入主畫面」
- Android：用 Chrome 開網址 → 右上 ⋮ → 「安裝應用程式」或「新增至主畫面」

這樣在手機上就會有獨立圖示，看起來很像 App。

**優點**：完全免費、幾分鐘就能完成。
**缺點**：是網頁版（不是原生 App）。

### 方法 B: 其他免費/低成本主機

- **Render.com**（推薦）
- **Railway.app**
- **Hugging Face Spaces**
- **PythonAnywhere**（免費版較慢）

## 4. 打包成可直接下載的程式（進階）

想讓用戶不用開瀏覽器就能用，有幾種方式：

### 選項 1: 使用 PyInstaller 打包（不推薦）
Streamlit 很難打包成單一執行檔，因為它需要啟動伺服器。

### 選項 2: 使用 Tauri + Streamlit（進階）
把網頁包成桌面 App（Windows / macOS / Linux）。

### 選項 3: 使用 Capacitor 把網頁包成手機 App（推薦真實 App 路線）

這是目前把 Streamlit 變成「真手機 App」最可行的方式。

步驟概要：
1. 先把 App 部署到網路上，拿到一個 https 網址
2. 用 Capacitor 建立一個空專案
3. 把你的網址當成 WebView 載入
4. 打包成 .ipa（iOS）和 .apk（Android）

優點：可以上架 App Store / Google Play

缺點：
- 需要學一些前端工具（Node.js、Capacitor）
- iOS 上架需要 Apple Developer 帳號（一年 $99 美金）
- Android 需要 Google Play Console（一次性 $25）

## 5. 真的要上架到 App Store / Google Play 商店（現實說明）

### 重要現實：

**純 Streamlit App 無法直接上架到 Apple App Store 或 Google Play。**

原因：
- App Store 只接受「原生」或特定框架打包的應用程式
- Streamlit 是網頁技術（Python + HTML）

### 目前實際可行的路線：

| 路線 | 難度 | 成本 | 推薦度 | 說明 |
|------|------|------|--------|------|
| **加到主畫面 (PWA)** | 極低 | 免費 | ★★★★★ | 最推薦現在做 |
| **用 Capacitor 包裝** | 中 | iOS $99/年<br>Android $25 | ★★★★ | 可以上架，但還是包 WebView |
| **用 Flutter 重寫** | 高 | 同上 | ★★★ | 體驗最好，但要全部重寫 |
| **用 React Native 重寫** | 高 | 同上 | ★★★ | 同上 |

## 用戶資料到底放在哪裡？（公開分享到 Threads 時必看）

這是你最需要搞清楚、也最容易被問的問題。

### 目前情況（使用 Streamlit Cloud 部署）

| 項目             | 儲存位置                  | 是否永久保存 | 誰能看到          |
|------------------|---------------------------|--------------|-------------------|
| 姓名 + 偏好     | 使用者瀏覽器 (localStorage) | 部分保留    | 只有該使用者     |
| 聊天記錄        | 使用者瀏覽器 (session)   | 不保存       | 只有該使用者     |
| 你的問題內容    | 不上傳伺服器永久保存     | -            | 開發者看不到     |
| SuperGrok 回答  | 在 grok.x.ai 產生         | 看 Grok 記錄 | 只有使用者       |

**總結：**
- 這是**極度隱私導向**的設計。
- 你（作為開發者）**看不到任何人的對話內容**。
- 使用者在不同裝置或清除資料後，記錄會消失。
- 這對 Threads 分享來說是優點（隱私好，不容易被投訴）。

### 在 App 裡我已經加上說明

在側邊欄有一個可展開的「📌 關於資料儲存與隱私」區塊，會清楚告訴使用者目前資料只存在他們的瀏覽器。
