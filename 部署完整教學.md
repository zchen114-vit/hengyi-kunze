# 洞察易生的經歷 - 完整部署教學（Windows 使用者專用）

這份文件教你**從零開始**把 App 部署到網路上，讓大家可以透過 Threads 連結使用。

## 前置準備

- 你已經有這個資料夾：`D:\Grok\hengyi-kunze`
- 你有 GitHub 帳號（沒有就去註冊）
- 你有 Streamlit Community Cloud 帳號（用 GitHub 登入即可）

---

## 步驟 1：把專案上傳到 GitHub

### 1.1 在 GitHub 建立新倉庫
1. 打開 https://github.com
2. 點右上角 **+** → **New repository**
3. Repository name 填：`hengyi-kunze` （或你喜歡的名字）
4. 設定為 **Public**（免費部署需要公開）
5. **不要**勾 "Add a README file"
6. 點 **Create repository**

### 1.2 上傳檔案（最簡單方式 - 使用 GitHub Desktop 或命令列）

#### 推薦方式 A：使用命令列（PowerShell）

打開 PowerShell，執行以下指令：

```powershell
# 切換到你的專案資料夾
cd D:\Grok\hengyi-kunze

# 初始化 git（如果還沒做）
git init

# 加入所有檔案
git add .

# 第一次 commit
git commit -m "第一次上傳 洞察易生的經歷 App"

# 把本機連接到 GitHub（把下面 YOUR_GITHUB_USERNAME 換成你的）
git remote add origin https://github.com/YOUR_GITHUB_USERNAME/hengyi-kunze.git

# 上傳
git branch -M main
git push -u origin main
```

#### 方式 B：使用 GitHub Desktop（圖形介面，比較不會出錯）
1. 下載安裝 GitHub Desktop
2. 登入你的 GitHub
3. 選擇 "Add an existing repository from your hard drive"
4. 選取 `D:\Grok\hengyi-kunze`
5. 建立新 repository 或連結剛剛建立的
6. 按 Publish

---

## 步驟 2：部署到 Streamlit Community Cloud

1. 打開瀏覽器，前往：https://share.streamlit.io
2. 用你的 GitHub 帳號登入
3. 點右上角藍色按鈕 **New app**
4. 在 "Repository" 欄位，選擇你剛剛建立的 `hengyi-kunze`
5. Main file path 保持預設 `app.py`
6. 按 **Deploy!**

等待 1-3 分鐘，部署完成後你會看到一個網址，例如：
`https://zchen114-vit-hengyi-kunze.streamlit.app`

**記下這個網址！** 這就是你要分享給大家的連結。

## 步驟 3：測試流程（請你現在跑一遍）

請你現在實際操作一次（很重要！）：

1. 用手機或另一個分頁打開上面拿到的網址
2. 在側邊欄輸入你的姓名（例如「阿哲」）和選擇偏好
3. 點擊一個分區（例如「感情與人際」）
4. 輸入一個測試問題，例如：「最近感情有點卡，該怎麼辦？」
5. 直接輸入問題
6. 易經顧問以傳統智慧回應
7. 可以繼續追問或換分區
8. 測試匯出對話功能
9. 試試「回到首頁」和換其他分區
10. 試試側邊欄的「匯出目前分區對話」
11. 試試首頁的「想讓我親自幫你解讀？」功能

做完後告訴我哪裡卡住、想調整，或哪個步驟不清楚。

## 步驟 4：讓網址比較穩定（選做）

免費 Streamlit Cloud 的缺點：
- 沒人使用 30~60 分鐘後會自動休眠（sleep）
- 下次有人點連結會等 10-30 秒才醒來

**想更穩定** 可以後續改用：
- Render.com（有免費方案，sleep 時間較好）
- Railway.app（如果想換更好主機）
- 自訂域名 + 付費方案

目前先用 Streamlit Cloud 就好。

## 常見部署問題

**Q: 上傳 GitHub 時說有錯誤？**
A: 通常是因為有 `.git` 衝突。刪除資料夾裡的 `.git` 資料夾再重新做。

**Q: Streamlit 部署失敗？**
A: 檢查 requirements.txt 是否只有 `streamlit>=1.32.0`

**Q: 網址打不開？**
A: 可能是 App 在 sleep，重新整理幾次，或等 30 秒。
